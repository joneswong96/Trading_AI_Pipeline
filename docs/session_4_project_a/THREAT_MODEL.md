## Candid security conclusion

OpenClaw is suitable only as a message/session/model operator behind the
deterministic Python boundary. Official documentation says sandboxing is off by
default, the Gateway remains on the host, workspace alone is not a security
boundary, and the sandbox is not perfect. Therefore the absent installed version
cannot yet be declared capable of enforcing this design. A pinned installation,
effective-policy inspection, canary tests, OAuth login, and Telegram pairing are
Jones-operated release gates; until then only offline/mocked review is enabled.
