# TradingView Project A shadow checklist

Use the isolated development chart only. No production credential or live webhook
is required.

## Install and create the alert

1. Open the approved isolated TradingView development chart.
2. Select `ICMARKETS:XAUUSD` and the 1-minute timeframe.
3. Open Pine Editor, paste `indicators/pine/snr_dashboard_project_a_v1.pine`, save
   it as `SNR Dashboard Project A v0.4.0 Shadow`, compile, and add it to the chart.
4. Compare the legacy title, plots, support/resistance levels, background colours,
   4-by-29 table, input defaults, and drawings with a baseline screenshot while
   `Enable Project A shadow alerts` remains off.
5. Turn on `Enable Project A shadow alerts`. Leave the expiry default at 30 bars
   unless explicitly testing expiry.
6. Create an alert and select this indicator's `Any alert() function call`.
7. Choose `Once Per Bar Close`. The script itself also calls
   `alert.freq_once_per_bar_close`; closed-bar evidence avoids open-bar repaint and
   matches the 1m Event 0.2 occurrence time. Do not choose Once Per Bar.
8. Leave the message field unchanged/generated by the `alert()` call. Do not paste
   a static JSON template over it. Use a local inspection destination or the alert
   log; do not configure a live or production webhook.

## Inspect evidence

1. Copy a generated JSON message from the alert log and parse it with a JSON
   viewer. Confirm `schema_version` is `0.2`, symbol/timeframe are XAUUSD/1m,
   `mode` is `SHADOW`, environment is `MT5_DEMO`, and `live_execution` is false.
2. Confirm `occurred_at`, `received_at`, payload `bar_time`, and `created_time` end
   in `Z`. A lower-timeframe evidence time, when present, is separately labelled.
3. Confirm repeated events in one lifecycle retain `setup_id` and `correlation_id`,
   distinct events have distinct `event_id`, and `causation_id` links to the prior
   event. Telemetry without an active setup may have a null setup ID.
4. Recompute `source.payload_hash` from the payload using the Event 0.2 canonical
   JSON rules before promoting captured evidence.

## Scenario tests

1. Rejection: replay a clean expansion into an eligible SNR followed by a closed
   1m rejection, sweep/reclaim, or engulf. Expect SETUP_CANDIDATE and then
   ANALYSIS_READY / SNR_REJECTION_READY.
2. Break: replay a first closed cross beyond the ATR buffer with body/range at
   least 0.55 and at least two continuing momentum slots. Expect
   ANALYSIS_READY / SNR_BREAK_READY even when the optional 5s arrow is null.
3. Invalidation: after readiness, close beyond the opposite structural boundary.
   Expect LIFECYCLE / SETUP_INVALIDATED, STRUCTURAL_BREAK, and the original setup
   ID.
4. Expiry: allow the configured observation window to elapse without closure.
   Expect LIFECYCLE / SETUP_EXPIRED, EXPIRED, and the original setup ID.
5. Range middle: replay an otherwise plausible reaction while HPA is range-middle.
   Confirm no ANALYSIS_READY message appears; telemetry may report RANGE_MIDDLE.
6. Duplicate suppression: force a chart recalculation without changing the closed
   bar or evidence. Confirm no duplicate alert. Then introduce a genuine new
   reaction or break and confirm it is not delayed by a fixed cooldown.

## Manual visual regression comparison

Capture before/after screenshots at the same symbol, timeframe, zoom, bar, theme,
and input values. Compare plots, bands, lines/boxes, labels, table content and
position, colours, resource-limit errors, historical bars, and the realtime bar.
Repeat once with the Project A flag off and once on; enabling alerts must not add
or alter chart drawings.

To disable the path, turn off `Enable Project A shadow alerts` and delete or pause
the corresponding TradingView alert. This does not affect legacy visuals.
